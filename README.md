# Ada's Cafe' — QR Self-Ordering System
## ระบบสั่งอาหารด้วย QR Code (Halal-Certified Cafe, Chiang Rai)

ระบบสั่งอาหารด้วยตนเองผ่านการสแกน QR Code (Bilingual TH/EN) สำหรับลูกค้า และหน้าจอแดชบอร์ดรับออเดอร์สำหรับพนักงาน (Waiter Screen) ทำงานแบบออฟไลน์ 100% ภายในเครือข่าย LAN ท้องถิ่น โดยไม่มีความต้องการในการเชื่อมต่ออินเทอร์เน็ต

---

## 🚀 วิธีเริ่มใช้งานที่ร้าน (Go-Live Guide)

### ขั้นตอนที่ 1 — ตั้งค่า IP Network
1. เปิด CMD บนเครื่อง PC ที่รัน Server -> พิมพ์ `ipconfig`
2. จดค่า **IPv4 Address** ของ WiFi หรือ LAN adapter (เช่น `192.168.1.100`)
3. ปิดเครื่องพิมพ์ครัว -> กดปุ่ม **Feed** ค้างไว้ -> เปิดเครื่องพิมพ์ -> ปล่อยปุ่ม Feed
4. จดค่า **IP Address** ที่ปรากฏบนใบ self-test (เช่น `192.168.1.150`)
5. คัดลอกและสร้างไฟล์คอนฟิกในโฟลเดอร์ `/server`:
   - Windows: `copy .env.example .env`
   - macOS/Linux: `cp .env.example .env`
6. แก้ไขไฟล์ `server/.env` โดยระบุค่า IP ที่จดไว้:
   ```env
   BRIDGE_SERVER_IP=192.168.1.xxx   # IP ของเครื่อง PC
   KITCHEN_PRINTER_IP=192.168.1.xxx # IP ของเครื่องพิมพ์ครัว
   ```

### ขั้นตอนที่ 2 — Build Frontend
คอมไพล์โปรเจ็กต์ React เพื่อเตรียมพร้อมสำหรับการรันระบบจริง:
```bash
cd client
npm install
npm run build
```

### ขั้นตอนที่ 3 — สร้าง QR Codes ของแต่ละโต๊ะ
เจนเนอเรท QR codes และใบ A4 พร้อมพิมพ์สำหรับทุกโต๊ะ (1–32):
```bash
cd ../scripts
npm install
node generate-qr.js
```
- เปิดไฟล์ `scripts/print-indoor.html` ในเบราว์เซอร์ -> กด **Ctrl+P** -> พิมพ์และนำไปติดตั้งที่ **โต๊ะ 1–20 (อินดอร์)**
- เปิดไฟล์ `scripts/print-outdoor.html` ในเบราว์เซอร์ -> กด **Ctrl+P** -> พิมพ์และนำไปติดตั้งที่ **โต๊ะ 21–32 (เอาท์ดอร์)**

### ขั้นตอนที่ 4 — รัน Server
```bash
cd ../server
npm install
node server.js
```
- สังเกต **Startup Checklist** บน terminal
- ตรวจสอบให้แน่ใจว่าไม่มีสัญลักษณ์ **✗** หรือคำเตือน **⚠️** เกี่ยวกับ Placeholder IP ก่อนเปิดให้บริการจริง

### ขั้นตอนที่ 5 — เปิด Waiter Screen บน Sunmi D3 Pro
1. เปิด Web Browser บนเครื่อง Sunmi D3 Pro (ที่เชื่อมต่อวง LAN เดียวกัน)
2. ไปที่ URL: `http://{BRIDGE_SERVER_IP}:3000/waiter` (แทนค่าด้วย IP ของเครื่อง Server)
3. กดปุ่มสีเขียว **"แตะเพื่อเปิดการแจ้งเตือนเสียง" / "Tap to Enable Sound Alerts"** (เนื่องจากนโยบายเบราว์เซอร์จะไม่ส่งเสียงแจ้งเตือนอัตโนมัติหากไม่มีการโต้ตอบจากผู้ใช้)
   *หมายเหตุ: ค่าสถานะการอนุญาตเสียงจะถูกรีเซ็ตทุกครั้งที่มีการโหลดหน้าใหม่ (Page Reload)*

### ขั้นตอนที่ 6 — ทดสอบระบบก่อนเปิดร้าน
1. เปิดเบราว์เซอร์บนสมาร์ทโฟนของท่าน (เชื่อมต่อ WiFi เดียวกันกับ Server)
2. ไปที่ URL: `http://{BRIDGE_SERVER_IP}:3000/order?table=1`
3. สั่งกาแฟหรือเมนูอาหาร 1 รายการ และกดยืนยันออเดอร์
4. **ตรวจสอบผลลัพธ์**:
   - [ ] เครื่องพิมพ์ครัวพิมพ์ใบ KOT ออกมาอย่างถูกต้อง (มีโลโก้ฮาลาลและรายการสินค้าภาษาไทย)
   - [ ] หน้าจอ Waiter Screen แสดงออเดอร์ใหม่พร้อมเสียงสัญญาณ Chime แจ้งเตือน
   - [ ] เมื่อ waiter กดปุ่ม "✓ รับออเดอร์" บัตรรายการจะเปลี่ยนเป็นแถบสีเขียว และใบครัวจะพิมพ์ซ้ำระบุสถานะยืนยัน

## 🛠 การแก้ไขปัญหา (Troubleshooting)

| อาการ (Issue) | สาเหตุที่อาจเกิดขึ้น (Possible Cause) | วิธีแก้ไข (Resolution) |
|---|---|---|
| **ใบครัวไม่พิมพ์ออกเครื่องพิมพ์** | IP ของเครื่องพิมพ์ครัวเปลี่ยน หรือไม่ได้เสียบสาย LAN | ตรวจสอบ IP บนใบ self-test และแก้ไขค่า `KITCHEN_PRINTER_IP` ใน `server/.env` |
| **ไม่มีเสียงแจ้งเตือนออเดอร์ใหม่** | Waiter Screen โหลดหน้าใหม่หรือไม่ได้กดปุ่มอนุญาตเสียง | แตะปุ่ม "แตะเพื่อเปิดการแจ้งเตือนเสียง" บนหน้าจอเพื่อเริ่มการทำงานใหม่ |
| **สแกน QR Code แล้วหน้าเว็บไม่โหลด** | เครื่องคอมพิวเตอร์ server ไม่ได้เปิดอยู่ หรือมือถือไม่ได้ต่อวง WiFi เดียวกัน | ตรวจสอบว่า server รันอยู่ และมือถือเชื่อมต่อ WiFi วง LAN ภายในร้าน |
| **ตัวอักษรไทยบนสลิปครัวอ่านไม่รู้เรื่อง** | เครื่องพิมพ์ไม่ได้ตั้งค่า Codepage เป็น CP874 | ตรวจสอบคู่มือเครื่องพิมพ์ และตรวจสอบว่า Codepage 26 (TIS-11) เปิดใช้งานแล้วในตัวเครื่องพิมพ์ |
